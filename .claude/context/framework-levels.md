# Context: Framework Maturity Levels

## How to assess what stage this framework is at for a given project or across the portfolio.

### Stage 1: Capture (< 10 entries)
- Focus on running `/log-session` after substantive work
- Don't mine yet — not enough data for patterns
- Establish the habit of logging

### Stage 2: Identify (10-30 entries)
- Run `/mine-skills` per project
- Spot local patterns (task sequences, context loads, friction)
- Extract first 2-3 primitives

### Stage 3: Extract (30-50 entries, 2+ projects)
- Run `/meta-mine` for cross-project patterns
- Identify universal vs. domain-specific primitives
- Build first composed skills from primitives

### Stage 4: Compose (5+ primitives extracted)
- Skills reference primitives explicitly
- Context slices reduce ramp time measurably
- Coverage score tracks framework utility

### Stage 5: Mature (70%+ coverage, 3+ projects)
- Framework is self-sustaining
- Mining finds edge cases, not major gaps
- New projects bootstrap from framework starter kit

## Signals You're Ready to Advance

| From → To | Signal |
|-----------|--------|
| 1 → 2 | You've logged 10+ sessions and notice yourself writing similar entries |
| 2 → 3 | You have 3+ primitives and a second project generating journal entries |
| 3 → 4 | Mining reports show > 50% of steps are covered by existing primitives |
| 4 → 5 | New projects reach productive state in < 1 session using framework bootstrap |
