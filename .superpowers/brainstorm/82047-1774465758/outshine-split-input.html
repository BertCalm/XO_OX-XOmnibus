C — Drop-Dominant with Collapsed Browser

Drop zone takes 85% of the space (matches the empty state spec's 80% negative space principle). Browse and Recent are buttons in a bottom bar that expand into an overlay when clicked. Grain chips accumulate next to the buttons. Simplest layout — most space for the primary action.

My Recommendation

A (Vertical Split) — because you said you'll use both constantly. The split means you never switch modes. The browser is always there when you need to navigate to a folder. The drop zone is always there when Finder is already open. The "Add Selected →" button makes it clear how files move from browser to pipeline.

B is good if screen real estate is tight. C is good for casual users who mostly drag-drop. But for a daily-driver power user with samples organized in deep folder hierarchies on an MPC-connected Mac, the always-visible browser is the right call.
