## 3. Recording Specifications

### Primary (WAV, archival)
| Parameter | Value | Rationale |
|-----------|-------|-----------|
| Sample rate | 44100 Hz | Web standard; all browser AudioContext defaults |
| Bit depth | 24-bit | Archival quality; gives headroom for normalization |
| Channels | Stereo | All engines have stereo output |
| Duration | 12–15 seconds | Long enough for pad tails + attack; short enough to stay focused |
| Loudness target | -14 LUFS integrated | Streaming/web standard (matches Spotify, YouTube) |
| True peak ceiling | -1.0 dBTP | Headroom before lossy re-encode |

### Web-delivery (converted from WAV)
| Format | Bitrate | Notes |
|--------|---------|-------|
| MP3 | 192 kbps CBR | Broadest browser support |
| OGG Vorbis | Q6 (~192 kbps) | Better quality/size ratio; Firefox primary |
| WebM/Opus | 128 kbps | Smallest file; Chrome/Edge |

For each engine: 1 archival WAV + 1 MP3 + 1 OGG.

## 4. What to Play — Per Engine Type

Match the phrase to what the engine does, not to what is easy:

| Engine Type | What to Play | Duration Notes |
|-------------|--------------|----------------|
| **Pad engines** (Opal, Drift/Odyssey, Morph/OddOscar, Ombre, Overcast, Overflow, Overwash, Overworn, Overlap) | Single lush chord (Cmaj7 or Am9), hold 8–10s, let tail ring | Capture full release tail |
| **Bass engines** (OceanDeep, Ogre, Olate, Omega, Obese/Fat) | Single low note (C2), then walking root pattern C–G–F–Bb over 8s | Show both sustain and movement |
| **Drum/percussion** (Onset, Offering, Ostinato, Oware) | 4-bar pattern at ~90 BPM — let the internal sequencer/patterns run | Use built-in rhythm if available |
| **Rhodes/EP/Piano** (Oasis, Oddfellow, Ochre, Onkolo, Opcode, Oven, Opaline, Obelisk) | Simple 4-bar chord melody (Cmaj7–Am7–Fmaj7–G7) at ~80 BPM | Show voicing and touch |
| **Organ engines** (Octave, Oleg, Otis, Oto) | Held chord (C–E–G–Bb), then release and sustain, swell with mod wheel | Show drawbar character |
| **Lead/melodic synths** (Bite/Overbite, Oblique, Origami, Obrix, Ouie, OpenSky, Optic, Oracle, Orbital, Overtone) | 8-bar melodic phrase, moderate tempo ~100 BPM | Show full expression range |
| **Strings/plucked** (Orchard, Osier, Overgrow, Oxalis, Orphica, Ohm) | Arpeggiated Am–G–F–E pattern, 2 octaves, let sympathetic rings sustain | Show material character |
| **Brass** (Ottoni) | Swell from pp to ff over 4 bars, then a tight staccato phrase | Show dynamics |
| **Woodwind** (Obbligato) | Single melodic phrase, let both voices enter and respond to each other | Show dual-voice dialogue |
| **Wavetable/complex** (Orca, Octopus, Organon, Organism, Obscura, Obsidian, Ouroboros, Orbweave) | Hold chord and sweep COUPLING macro slowly through full range | Show evolution |
| **Adaptive/special** (Ocelot, Outwit, Ombre, Osmosis, Owlfish) | 8 bars with slow filter + macro sweep, let engine evolve | Show biome/arc changes |
| **World percussion** (Ole, Ostinato, Oware) | Natural groove pattern relevant to the engine's cultural identity | Show rhythmic character |
| **Chip/retro** (Overworld) | Classic NES-style arpeggio, then melodic phrase using ERA macro | Show three eras |
| **Experimental** (Optic, Opera) | Opera: hold a note and let Conductor arc complete one full dramatic cycle | Show autonomous arc |

## 5. Step-by-Step Capture Workflow

### Setup (do once)
1. Open XOmnibus in standalone mode (or Logic Pro AU)
2. Set audio output: your interface (recording at 44.1 kHz / 24-bit)
3. Set MIDI input: your keyboard or controller
4. Open DAW session at 44100 Hz, 24-bit — create one stereo audio track
5. Set input monitoring: XOmnibus stereo out → DAW stereo track
6. Create a folder: `~/Documents/xoox-hero-clips/wav/`
7. Verify the 71-filename list is ready (Section 2 above)

### Per-Engine Capture (repeat 71 times)
```
For each engine:

a. In XOmnibus — select the engine slot
b. Navigate to the hero preset from Section 1
   (If preset not yet created: load a named init patch or manually dial in
    the character described in the Hero Preset column)
c. Set mod wheel to 0, pitch bend to center, aftertouch off
d. Play the appropriate phrase from Section 4 (12–15 seconds total)
e. Record: press record in DAW, play phrase, let tail ring fully
f. Stop recording
g. Trim: remove silence before attack (0.1s pre-roll OK)
h. Leave full tail — do not fade artificially
i. Export mono-named render as 24-bit / 44100 Hz WAV
j. Name the file exactly per Section 2 (e.g., `oasis-hero.wav`)
k. Save to `~/Documents/xoox-hero-clips/wav/`
l. Move to next engine
```

### Session Pacing
- 71 engines at ~4 minutes each (play, record, trim, export) = ~5 hours
- Recommended: do in batches of 15–20 engines per session
- Suggested batch splits by aquatic zone:
  - Batch 1: Sunlit Shallows + Surface (engines 1, 2, 3, 11, 12, 18, 19, 26, 37, 45, 46, 54, 65, 71)
  - Batch 2: Open Water + Reef (engines 4, 5, 7, 8, 9, 14, 15, 20, 21, 24, 27, 30, 31, 34, 35, 38, 39, 40, 49, 50, 51)
  - Batch 3: Thermocline + Deep (engines 6, 10, 16, 17, 25, 28, 33, 36, 41, 42, 47, 48, 52, 53, 55, 57, 58, 61, 62, 63, 68, 69, 70)
  - Batch 4: Abyss + Kitchen/Cellar/Garden/Broth/Fusion (remaining 23 engines)

## 6. Post-Processing Script

Requires: `ffmpeg` (already installed at `/opt/homebrew/bin/ffmpeg`).
Save as `~/Documents/xoox-hero-clips/process.sh`.

```bash
#!/bin/bash
# XO_OX Hero Clip Post-Processor
# Requires: ffmpeg (brew install ffmpeg)
# Normalizes WAVs to -14 LUFS, then converts to MP3 + OGG for web delivery.

set -e

WAV_DIR="$HOME/Documents/xoox-hero-clips/wav"
MP3_DIR="$HOME/Documents/xoox-hero-clips/mp3"
OGG_DIR="$HOME/Documents/xoox-hero-clips/ogg"
NORM_DIR="$HOME/Documents/xoox-hero-clips/wav-normalized"

mkdir -p "$MP3_DIR" "$OGG_DIR" "$NORM_DIR"

FFMPEG="/opt/homebrew/bin/ffmpeg"

echo "=== XO_OX Hero Clip Post-Processor ==="
echo "Source WAVs: $WAV_DIR"
echo ""

for wav in "$WAV_DIR"/*.wav; do
  base=$(basename "$wav" .wav)
  echo "Processing: $base"

  # Step 1: Loudness normalize to -14 LUFS, -1.0 dBTP ceiling
  norm="$NORM_DIR/${base}.wav"
  "$FFMPEG" -y -i "$wav" \
    -af "loudnorm=I=-14:TP=-1.0:LRA=11" \
    -ar 44100 -c:a pcm_s24le \
    "$norm" 2>/dev/null
  echo "  Normalized: $norm"

  # Step 2: Convert to MP3 (192 kbps CBR)
  mp3="$MP3_DIR/${base}.mp3"
  "$FFMPEG" -y -i "$norm" \
    -codec:a libmp3lame -b:a 192k -ar 44100 \
    "$mp3" 2>/dev/null
  echo "  MP3: $mp3"

  # Step 3: Convert to OGG Vorbis (Q6)
  ogg="$OGG_DIR/${base}.ogg"
  "$FFMPEG" -y -i "$norm" \
    -codec:a libvorbis -qscale:a 6 -ar 44100 \
    "$ogg" 2>/dev/null
  echo "  OGG: $ogg"

  echo "  Done: $base"
  echo ""
done

echo "=== Verification ==="
echo "WAV count (original):   $(ls "$WAV_DIR"/*.wav 2>/dev/null | wc -l | tr -d ' ')"
echo "WAV count (normalized): $(ls "$NORM_DIR"/*.wav 2>/dev/null | wc -l | tr -d ' ')"
echo "MP3 count:              $(ls "$MP3_DIR"/*.mp3 2>/dev/null | wc -l | tr -d ' ')"
echo "OGG count:              $(ls "$OGG_DIR"/*.ogg 2>/dev/null | wc -l | tr -d ' ')"
echo ""
echo "Expected: 71 files in each directory."

# Check for missing files
echo ""
echo "=== Missing File Check ==="
ENGINES=(overbite oblong odyssey overdub obese oddoscar oaken oasis obbligato obelisk
  oblique obrix obscura obsidian oceandeep oceanic ocelot ochre octave octopus
  oddfellow offering ogre ohm olate ole oleg ombre omega onkolo onset opal opaline
  opcode opensky opera optic oracle orbital orbweave orca orchard organism organon
  origami orphica osier osmosis osprey osteria ostinato otis oto ottoni ouie
  ouroboros outwit oven overcast overflow overgrow overlap overtone overwash overworld
  overworn oware owlfish oxalis oxbow oddfelix)

MISSING=0
for eng in "${ENGINES[@]}"; do
  if [ ! -f "$MP3_DIR/${eng}-hero.mp3" ]; then
    echo "  MISSING MP3: ${eng}-hero.mp3"
    MISSING=$((MISSING + 1))
  fi
done

if [ $MISSING -eq 0 ]; then
  echo "  All 71 MP3 files present."
else
  echo "  $MISSING MP3 file(s) missing."
fi
```

Make executable:
```bash
chmod +x ~/Documents/xoox-hero-clips/process.sh
```

Run after recording all 71 WAVs:
```bash
~/Documents/xoox-hero-clips/process.sh
```
