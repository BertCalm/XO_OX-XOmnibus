### INF-03 [P1] — `FamilyWaveguideTest` has no `target_include_directories` or `target_link_libraries`

**File:** `CMakeLists.txt` (lines ~260-266)

```cmake
add_executable(FamilyWaveguideTest
    Tests/FamilyWaveguideTest.cpp
)
# No target_include_directories — Source/ not on its include path
# No target_link_libraries — will fail to link
```

`Tests/FamilyWaveguideTest.cpp` includes `"../Source/DSP/FamilyWaveguide.h"` via a relative path hack that works only from the `Tests/` directory. The CMake target has no include path set, so `cmake --build` will fail for this target. `XOceanusTests` has the correct include setup but `FamilyWaveguideTest` was missed.

**Fix:**
```cmake
target_include_directories(FamilyWaveguideTest PRIVATE
    "${CMAKE_CURRENT_SOURCE_DIR}/Source"
)
```

### INF-04 [P2] — `_D_FORTIFY_SOURCE=2` applied unconditionally in Debug builds

**File:** `CMakeLists.txt` (security hardening block)

`-D_FORTIFY_SOURCE=2` requires `-O1` or higher to function. GCC 12+ emits a warning when `FORTIFY_SOURCE` is defined without optimization. The current code applies it to all build types including Debug (where optimization is typically `-O0`).

**Fix:** Guard it:
```cmake
if(NOT CMAKE_BUILD_TYPE STREQUAL "Debug")
    add_compile_options(-D_FORTIFY_SOURCE=2)
endif()
```
