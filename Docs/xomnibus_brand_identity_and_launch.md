## 1. Brand Foundation

### 1.1 What XOmnibus Is

XOmnibus is a free, open-source multi-engine synthesizer platform that combines 6 character instruments into one unified creative environment. It is built by XO_OX Designs — a solo developer/sound designer operation that prioritizes character over feature count.

**It is not:**
- A commercial product
- A DAW replacement
- A plugin that tries to do everything

**It is:**
- A creative tool that unlocks sounds impossible with single-engine synths
- A platform where engines couple, collide, and mutate into new instruments
- An open invitation for producers, sound designers, and developers to build on

### 1.2 Brand Values

| Value | Meaning | How It Shows Up |
|-------|---------|-----------------|
| **Character** | Every sound has personality | 1000 presets with evocative names, not "Init Patch 47" |
| **Coupling** | Forces that shouldn't work together create something new | Cross-engine modulation is the signature feature |
| **Openness** | Free tools, shared knowledge, no gatekeeping | Open source, community presets, documented architecture |
| **Craft** | Small-batch quality over mass-market quantity | Each engine was a standalone instrument first |
| **Duality** | X and O, tension and release, familiar and alien | The XO_OX name itself, the coupling philosophy |

### 1.3 Brand Voice

- **Confident, not arrogant.** "This sounds incredible" not "the most advanced synth ever built."
- **Warm, not corporate.** Write like a producer talking to another producer.
- **Specific, not vague.** "Kick pumps the pad engine, snare brightens the filter" not "innovative sound design capabilities."
- **Inviting, not exclusive.** "Load a preset, twist a macro, hear what happens" not "designed for professional sound designers."

## 2. Name & Wordmark

### 2.1 The Name

**XOmnibus** — Latin: "for all." The collected works of XO_OX Designs, accessible to everyone.

The name carries three meanings:
1. **Omnibus = "for all"** — open source, community-focused
2. **Omnibus = a collection** — 6 instruments in one platform
3. **XO** prefix — maintains the XO_OX naming convention (XOddCouple, XOverdub, XOdyssey, XOblongBob, XObese, XOnset)

### 2.2 Wordmark Treatment

```
Primary:        XOmnibus
Stylized:       XO|mnibus  (pipe separator between XO and mnibus)
Monogram:       XO (the brand constant across all instruments)
Full brand:     XO_OX · XOmnibus
```

**Typography:** Space Grotesk Bold for the wordmark. The geometric precision of Space Grotesk contrasts with the organic, character-driven sound design philosophy — that tension is intentional.

### 2.3 Pronunciation

"ex-OH-mnibus" — stress on the second syllable. Rhymes with "omnibus."

## 3. Logo System

### 3.1 Primary Mark: The Coupling Symbol

The XOmnibus logo is built from the brand's core concept: **two forces coupling to create something new.**

**Concept:** Two interlocking rings — one an X shape, one an O shape — connected by a visible bridge/link between them. The bridge represents coupling. When engines couple, the bridge glows gold.

```
Variations:

○ Logomark only       — The coupled X/O symbol (app icon, favicon, small sizes)
○ Logomark + wordmark — Symbol left, "XOmnibus" right (header bar, marketing)
○ Wordmark only       — "XOmnibus" in Space Grotesk Bold (tight spaces)
○ Full brand          — "XO_OX" above, coupling symbol, "XOmnibus" below (splash screen)
```

### 3.2 Engine Sub-Marks

Each engine gets a minimal icon that lives inside its panel and appears in preset browser badges. These are secondary to the XOmnibus mark — they identify the engine, not the product.

| Engine | Icon | Visual | Color |
|--------|------|--------|-------|
| XOddCouple | Two interlocking circles | The original XO coupling | Terracotta + Teal |
| XOverdub | Tape reel | Single circle with inner hub, tape threading | Olive |
| XOdyssey | Spiral path | Journey from center outward | Violet |
| XOblongBob | Rounded rectangle | The "oblong" — soft, tactile | Amber |
| XObese | Bold square | Heavy, industrial weight | Hot Pink |
| XOnset | Dot grid | Circuit/matrix precision | Electric Blue |

### 3.3 Logo Colors

```
Primary mark:       #1A1A1A on light backgrounds
                    #F8F6F3 on dark backgrounds
Coupling bridge:    #E9C46A (XO Gold) — always gold regardless of background
Monochrome:         Single color (black or white) for print/merch
```

### 3.4 Logo Clear Space

Minimum clear space around the logo = the width of the "O" in XOmnibus. No other elements should intrude into this space.

### 3.5 Logo Don'ts

- Never rotate the logo
- Never change the coupling bridge color (always gold)
- Never separate the X and O symbols
- Never use engine sub-marks as the product logo
- Never place on busy backgrounds without a backing panel

## 5. Visual Identity Beyond the Plugin

### 5.1 The Gallery Model

XOmnibus is a **gallery**. This isn't a metaphor — it's the organizing principle for every visual and community decision.

| Gallery Concept | XOmnibus Equivalent |
|-----------------|---------------------|
| Gallery walls | The warm white shell — header, preset browser, macro bar |
| Exhibition | An engine panel — colored, textured, unique to that instrument |
| Corridor between exhibitions | The coupling strip — golden bridge connecting two engines |
| Exhibition catalog | The preset browser — organized by mood, searchable by DNA |
| Guest exhibition | Community presets — credited to the artist, displayed in the same gallery |
| Opening night | Engine spotlight video — showcasing one engine's character |
| Curator's pick | "Start Here" and "Hero Presets" smart collections |
| Gallery guide | Getting Started walkthrough + engine deep-dives |
| Art breeding program | Preset breeding — two presets combine DNA to produce offspring |

This model scales from the plugin UI to the website to social media to documentation. Every touchpoint reinforces: **XO_OX is the gallery. XOmnibus is the exhibition space. Each engine is the art.**

### 5.2 Photography & Imagery Style

Since XOmnibus is software, "photography" means:
- **Screenshots:** Always light mode, retina resolution, showing a cross-engine preset with coupling active
- **Abstract textures:** Engine-colored gradients used as backgrounds for social posts (terracotta→teal for XOddCouple content, violet→cyan for XOdyssey content)
- **Waveform art:** Actual audio renders from XOmnibus presets, visualized as waveforms or spectrograms, used as hero images
- **No stock photos.** Everything is generated from the tool itself or designed from the brand palette.

### 5.3 Color Usage Outside the Plugin

```
Marketing backgrounds:    Warm white #F8F6F3 (primary) or near-black #1A1A1A (accent)
Call-to-action buttons:   XO Gold #E9C46A
Body text:                #1A1A1A on light, #EEEEEE on dark
Section accents:          Engine colors, used one at a time per section
Social media borders:     2px XO Gold on all images
```

### 5.4 Merch & Physical (If Applicable)

If community interest warrants physical goods:
- **Stickers:** Coupling symbol in gold on black or white
- **T-shirts:** Wordmark + "Couple everything." — clean, no busy graphics
- **Posters:** Mood Map visualization of the full preset library — abstract art that is actually data

### 6.2 Getting Started Guide Structure

1. **Download & install** (2 minutes)
2. **Load a hero preset** (auto-loaded on first launch)
3. **Twist the 4 macros** — hear what CHARACTER, MOVEMENT, COUPLING, and SPACE do
4. **Switch moods** — browse Foundation, Atmosphere, Entangled presets
5. **Activate a second engine** — hear coupling for the first time
6. **Find Similar** — use DNA search to discover related sounds
7. **Save your first preset**

The guide should take 10 minutes from download to saving a custom preset.

### 6.3 Engine Deep-Dive Template

Each engine gets a dedicated page:
- **Identity paragraph** — what this engine sounds like and why it exists
- **Signal flow diagram** — visual routing
- **Key parameters** — the 5-8 parameters that define this engine's character
- **Macro mapping** — what M1-M4 control in this engine
- **Coupling sweet spots** — which other engines pair well and why
- **5 recommended presets** — curated starting points

## 7. Open Source Strategy

### 7.1 License

**GPLv3** — ensures derivative works remain open source. This is the standard for open-source audio plugins (Vital, Surge, Dexed all use GPL variants).

### 7.2 Repository Structure

```
xomnibus/
├── Source/                    # C++ plugin source
├── Presets/                   # Factory .xometa files
├── Tools/                     # Python utilities (DNA, breeding, migration)
├── Docs/                      # Technical documentation
├── Assets/                    # Icons, fonts, images
├── Community/                 # Preset submission templates, contributor guide
├── README.md                  # Overview + quick start
├── CONTRIBUTING.md            # How to contribute code, presets, or docs
├── LICENSE                    # GPLv3
└── CLAUDE.md                  # AI development guide
```

### 7.3 Contribution Pathways

Not everyone contributes code. XOmnibus should welcome:

| Contribution Type | Barrier to Entry | Impact |
|-------------------|-------------------|--------|
| **Bug reports** | None — just use it | High |
| **Preset submissions** | Low — design a sound, export .xometa | High |
| **Preset packs** | Medium — curate 10-20 themed presets | High |
| **Documentation** | Low — writing skill | Medium |
| **Translations** | Low — language skill | Medium |
| **Engine development** | High — C++ / JUCE / DSP | Very high |
| **UI themes** | Medium — design + CSS/JUCE LookAndFeel | Medium |
| **Video tutorials** | Medium — production + video skill | High |

### 7.4 Community Preset Pipeline (Guest Exhibitions)

```
Sound designer creates preset ("the artist prepares a piece")
    ↓
Exports .xometa file ("frames the work")
    ↓
Submits via GitHub PR or community portal ("submits to the gallery")
    ↓
Auto-validation: schema check, DNA computation, name collision check ("gallery standards check")
    ↓
Human review: sonic quality, macro responsiveness, naming convention ("curation")
    ↓
Merged into Community Presets folder ("hung on the wall")
    ↓
Available in next release — credited to the designer ("opening night")
```

## 10. Dissemination: Making It Stick

### 10.1 Why Free Plugins Get Ignored

Most free plugins fail because:
1. They look cheap (dark UI, generic knobs)
2. The presets are afterthoughts
3. There's no story — just "here's a synth"
4. No community — download and forget

### 10.2 How XOmnibus Avoids This

| Problem | XOmnibus Solution |
|---------|-------------------|
| Looks cheap | Light-mode-first design, gallery aesthetic, professional typography |
| Bad presets | 1000 factory presets with DNA fingerprints, evocative names, macro responsiveness |
| No story | "Couple everything" — a clear, memorable concept |
| No community | Discord, preset submissions, community packs, contributor recognition |
| Hard to start | Auto-loads hero preset, 10-minute getting started guide |
| Overwhelming | Mood-based browsing, DNA similarity search, curated "Start Here" collection |

### 10.3 The 30-Second Hook

A new user should experience this in their first 30 seconds:

1. Open XOmnibus → hero preset auto-loads (two engines coupled)
2. Play a chord → hear something they've never heard before (coupling is audible)
3. Turn COUPLING macro to zero → hear the engines decouple
4. Turn it back up → "oh, THAT'S what coupling does"

If the first 30 seconds don't create a "what was that?" moment, the preset library needs work.

### 10.4 Shareability

Every interaction should produce something shareable:
- **Preset breeding** → "I bred Velvet Morning with Circuit Haze and got this" (screenshot of DNA radar + audio clip)
- **Mood Map** → "Look where my preset landed in the DNA space" (screenshot)
- **Coupling visualization** → "Two engines talking to each other" (screenshot/video)

Build share buttons into the plugin itself: "Export preset + DNA card as image" for social posting.

## 13. Brand Cohesion with Engine Character

### 13.1 The Challenge

Each engine was designed as a standalone instrument with its own visual identity. XOmnibus must unify them without erasing their personality.

### 13.2 The Solution: Shared Frame, Unique Canvas

**What stays consistent (the XOmnibus shell):**
- Warm white background (#F8F6F3)
- Space Grotesk / Inter / JetBrains Mono font stack
- XO Gold (#E9C46A) for macro knobs, active states, brand elements
- 4px spacing grid, 12px panel radius, 6px button radius
- Header bar layout, preset browser, mood tabs

**What changes per engine (the canvas inside the frame):**
- Accent color (terracotta, olive, violet, amber, hot pink, electric blue)
- Panel texture/material hint (brushed metal, wood grain, holographic, liquid glass, concrete, PCB)
- Knob style (subtle variations in arc thickness, indicator style)
- Icon set (engine-specific waveform and filter shapes)

### 13.3 Engine Character Preservation

| Engine | Preserved Character | Adapted to Platform |
|--------|--------------------|--------------------|
| **XOddCouple** | Dual-panel split (X and O sides), coupling visualization | Split becomes left/right within the engine frame |
| **XOverdub** | Warm analog feel, VU meter aesthetic | VU meters become subtle level indicators, cream tones in panel |
| **XOdyssey** | Psychedelic gradients, journey feel | Gradient appears as panel header accent, journey slider prominent |
| **XOblongBob** | Soft rounded corners, tactile warmth | Slightly more rounded controls within the engine panel |
| **XObese** | Bold, industrial, high contrast | Bolder labels, higher contrast within panel, concrete texture |
| **XOnset** | Grid precision, circuit traces | Grid-aligned controls, LED-style indicators |

### 13.4 Design Principle

> "The shell is the same gallery for every exhibition. The art inside changes completely."

When an engine loads, its panel should feel like stepping into that engine's world — but the gallery walls (header, preset browser, macros, coupling strip) never change. This creates recognition: users always know they're in XOmnibus, but they also know immediately which engine they're working with.
