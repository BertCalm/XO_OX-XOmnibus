## P34 Triage Update (2026-05-03)

**PR #1513 swept 10 P34 candidates on 2026-05-03 and resolved the pattern into two distinct mechanisms:**

- **P34a: sr-fallback bug** (e.g., `(sr > 0) ? sr : 44100`) — true hardcoded fallback when sample rate detection fails
- **P34b: buffer sizing sub-pattern** (e.g., `kMaxDelay = 4096` assuming 44.1 kHz max) — buffer allocated assuming ceiling of 44.1 kHz, overflows at 96 kHz

**Results:** 3 true P34 bugs (Outcrop P34a, Ostinato P34b, Owlfish P34b) + 7 false positives using correct sr-scaling idiom `srScale = srf / 44100.0f` where `srf` is already properly set. See individual engine notes below.
