*XO_OX Designs | March 2026*

## Executive Summary

XOptic is a zero-audio spectral analysis engine that already extracts 8 spectral features (bass/mid/high energy, centroid, flux, total energy, transient detection, AutoPulse) from any audio it analyzes. The Oxport pipeline's `xpn_cover_art.py` currently generates cover art from static per-engine visual style definitions (9 style functions, 44 engine color palettes).

This spec proposes connecting the two: XOptic analyzes rendered preset audio offline, exports a spectral fingerprint per preset, and `xpn_cover_art.py` consumes that fingerprint to generate artwork that is literally derived from the sound. Every pack cover becomes unique to its sonic content. Every preset can have its own visual identity.

## Part 1: What XOptic Already Knows

### Spectral Features Available (per-sample, real-time)

| Feature | Type | Range | What It Tells You |
|---------|------|-------|-------------------|
| `bandEnergy[0..7]` | 8 floats | 0-1+ | Energy in Sub, Bass, Lo-Mid, Mid, Hi-Mid, Presence, Brilliance, Air |
| `spectralCentroid` | float | 0-1 | Brightness (0 = dark/bassy, 1 = bright/airy) |
| `spectralFlux` | float | unbounded | Rate of spectral change (high = transient-rich, low = sustained) |
| `totalEnergy` | float | 0-1+ | Overall loudness/density |
| `transientDetect` | bool | 0 or 1 | Binary onset gate (flux > 0.05 threshold) |
| `autoPulseLevel` | float | 0-1 | Rhythmic pulse output (if enabled) |

### What This Means for Visual Generation

XOptic can answer these questions about any audio:
- **How bright is this preset?** (centroid)
- **How dense/thick is it?** (totalEnergy)
- **Is it percussive or sustained?** (flux, transient count)
- **Where does the energy live?** (8-band distribution)
- **How does it evolve over time?** (centroid/energy trajectories)

These map directly to visual properties: color temperature, density of visual elements, sharpness vs. softness, spatial distribution, and animation/flow.

## Part 2: The Spectral Fingerprint

### Definition

A **Spectral Fingerprint** is a compact JSON summary of a preset's sonic character, generated by running XOptic's analyzer over a rendered audio buffer. It captures both the static timbral identity and the temporal evolution.

### Data Format: `OpticFingerprint`

```json
{
  "version": 1,
  "engine": "ONSET",
  "preset_name": "808 Reborn",
  "duration_sec": 4.0,
  "sample_rate": 48000,

  "static": {
    "centroid_mean": 0.34,
    "centroid_std": 0.08,
    "energy_mean": 0.62,
    "energy_peak": 0.95,
    "flux_mean": 0.12,
    "flux_peak": 0.87,
    "transient_count": 14,
    "transient_density": 3.5,
    "band_energy_mean": [0.18, 0.25, 0.14, 0.11, 0.08, 0.04, 0.02, 0.01],
    "band_energy_peak": [0.45, 0.62, 0.38, 0.29, 0.22, 0.11, 0.06, 0.02]
  },

  "temporal": {
    "centroid_curve": [0.28, 0.31, 0.35, 0.38, 0.42, 0.39, 0.33, 0.30],
    "energy_curve":   [0.10, 0.85, 0.72, 0.55, 0.40, 0.30, 0.20, 0.12],
    "flux_curve":     [0.02, 0.87, 0.15, 0.08, 0.05, 0.03, 0.02, 0.01]
  },

  "derived": {
    "felix_oscar_polarity": 0.72,
    "warmth": 0.65,
    "aggression": 0.45,
    "movement": 0.38
  }
}
```

### Field Explanations

**`static`** captures the overall character across the full render:
- `centroid_mean/std` -- average brightness and how much it varies (std > 0.15 = evolving timbre)
- `band_energy_mean` -- the spectral shape, the "silhouette" of the sound
- `transient_density` -- transients per second (> 4 = percussive, < 1 = pad/drone)

**`temporal`** captures 8 equally-spaced snapshots across the render duration, giving the "arc" of the sound. These become the primary shape generators for visual art -- a preset that builds from silence to a peak produces a rising energy curve that maps to an ascending visual element.

**`derived`** maps spectral data to XO_OX brand concepts:
- `felix_oscar_polarity` -- 0.0 = pure Oscar (warm, dark, sustained, gill pink), 1.0 = pure feliX (bright, transient, electric, neon blue). Computed from: `centroid_mean * 0.4 + flux_mean * 0.3 + transient_density_normalized * 0.3`
- `warmth` -- bass-to-high ratio: `(band[0]+band[1]+band[2]) / (totalEnergy + epsilon)`
- `aggression` -- transient density + flux peak, normalized
- `movement` -- centroid_std + energy_std, normalized (how much the sound changes)

## Part 3: Generating the Fingerprint

### Offline Analysis Pipeline

The fingerprint is generated during the Oxport render phase, not in real-time. The workflow:

```
[Preset .xometa] --> [XOceanus OfflineAudioContext render]
                          |
                          v
                    [4-second stereo buffer at native sample rate]
                          |
                          v
                    [OpticBandAnalyzer processes every sample]
                          |
                          v
                    [Accumulate statistics per-sample]
                          |
                          v
                    [Write OpticFingerprint JSON to pack directory]
```

### Implementation: Python Wrapper

The fingerprint generator wraps XOptic's algorithm in pure Python (numpy), mirroring the C++ DSP exactly. This avoids needing to run the plugin headless.

```python
# xpn_optic_fingerprint.py — new file in Tools/

class OpticFingerprint:
    """Offline spectral fingerprint generator.

    Mirrors OpticBandAnalyzer's 8-band Cytomic SVF filter bank
    using scipy.signal biquad equivalents. Processes a rendered
    WAV file and outputs the fingerprint JSON.
    """

    CROSSOVER_FREQS = [80, 200, 500, 1000, 4000, 8000, 16000]
    BAND_NAMES = ["sub", "bass", "lo_mid", "mid", "hi_mid",
                  "presence", "brilliance", "air"]

    def analyze(self, audio_path: str) -> dict:
        """Analyze a rendered WAV and return OpticFingerprint dict."""
        ...

    def analyze_buffer(self, samples: np.ndarray, sr: int) -> dict:
        """Analyze a numpy buffer directly."""
        ...
```

The Python implementation uses `scipy.signal.sosfilt` with bandpass Butterworth filters at the same crossover frequencies as the C++ OpticBandAnalyzer. The envelope follower is a simple one-pole lowpass at 30 Hz. Results will be within ~1-2% of the C++ version -- close enough for visual generation, which is perceptual, not sample-accurate.

### When Fingerprints Are Generated

1. **During `xpn_render_spec.py` execution** -- after each preset's audio is rendered to WAV, the fingerprint analyzer runs on that WAV and writes a sidecar `.fingerprint.json`
2. **During `xpn_cover_art.py` execution** -- the cover art generator reads all fingerprint files from the pack directory and uses them to inform the visual output
3. **On-demand** -- `python xpn_optic_fingerprint.py --input rendered.wav --output fingerprint.json`

## Part 4: From Fingerprint to Art

### A. Spectral Fingerprint Art (per-pack)

The pack cover art incorporates the aggregate spectral character of all presets in the pack.

**How it works:**

1. Load all `.fingerprint.json` files from the pack
2. Compute the pack-level aggregate:
   - Average `band_energy_mean` across all presets = the pack's spectral silhouette
   - Average `centroid_mean` = the pack's overall brightness
   - Max `transient_density` = the pack's percussive character
   - Average `felix_oscar_polarity` = the pack's position on the feliX-Oscar axis
3. Use these to modulate the existing style functions in `xpn_cover_art.py`:

```python
def generate_cover_with_fingerprint(
    engine: str,
    pack_name: str,
    output_dir: str,
    fingerprints: list[dict],  # NEW: list of OpticFingerprint dicts
    ...
) -> dict:
    # Aggregate fingerprints
    agg = aggregate_fingerprints(fingerprints)

    # Modulate style parameters based on spectral content
    style_params = {
        "density":    agg["energy_mean"],        # Dense presets = dense visuals
        "sharpness":  agg["flux_mean"],           # Transient-heavy = sharp edges
        "warmth":     agg["warmth"],              # Warm = warm color shift
        "movement":   agg["movement"],            # Evolving = flowing elements
        "brightness": agg["centroid_mean"],        # Bright presets = bright accents
        "polarity":   agg["felix_oscar_polarity"], # feliX=blue, Oscar=pink
    }

    # The spectral silhouette becomes a literal visual element
    # 8 bars representing the band energy distribution
    draw_spectral_silhouette(arr, agg["band_energy_mean"], accent)

    # Temporal curves from the loudest preset become flowing lines
    draw_energy_arc(arr, agg["energy_curve"], accent)
```

**What changes visually:**

| Spectral Property | Visual Effect |
|-------------------|---------------|
| High centroid (bright preset) | Lighter background, more high-frequency visual detail |
| Low centroid (dark preset) | Darker background, fewer but larger visual elements |
| High transient density | Sharp edges, staccato visual elements, angular cuts |
| Low transient density | Smooth gradients, flowing curves, soft gaussian elements |
| High energy | Dense visual fill, higher opacity, more elements |
| Low energy | Sparse, minimalist, more negative space |
| feliX polarity > 0.5 | Color shifts toward Neon Tetra Blue `#00A6D6` |
| Oscar polarity > 0.5 | Color shifts toward Axolotl Gill Pink `#E8839B` |
| High movement | Flowing Lissajous-like curves, wide spatial spread |
| Low movement | Static geometric forms, centered composition |

### B. Spectral Silhouette Element

The 8-band energy distribution becomes a literal design element on the cover: a small spectral bar graph rendered in the engine's accent color, placed in a consistent position (bottom-right quadrant, above the XO_OX watermark). This is the "sonic fingerprint" -- visually distinct for every pack because it is derived from the actual audio content.

```python
def draw_spectral_silhouette(arr, band_means, accent, size, position="bottom_right"):
    """Draw 8-band spectral silhouette as a design element.

    Each bar's height = normalized band energy.
    Color = engine accent with brightness modulated by energy.
    Width = size * 0.15 (takes up ~15% of canvas width).
    """
    bar_width = int(size * 0.015)
    max_height = int(size * 0.12)
    gap = int(size * 0.004)

    # Position in bottom-right quadrant
    start_x = int(size * 0.72)
    base_y = int(size * 0.88)

    peak = max(band_means) if max(band_means) > 0 else 1.0

    for i, energy in enumerate(band_means):
        x = start_x + i * (bar_width + gap)
        bar_h = int((energy / peak) * max_height)
        brightness = 0.3 + 0.7 * (energy / peak)

        # Draw bar from base upward
        r, g, b = [c / 255.0 * brightness for c in accent]
        arr[base_y - bar_h : base_y, x : x + bar_width, 0] = r
        arr[base_y - bar_h : base_y, x : x + bar_width, 1] = g
        arr[base_y - bar_h : base_y, x : x + bar_width, 2] = b
```

### C. Color Extraction from Timbre

Instead of using only the engine's static accent color, the fingerprint modulates the color palette:

```python
def derive_palette_from_fingerprint(engine_accent, fingerprint):
    """Generate a 3-color palette from spectral analysis.

    Primary:   Engine accent (identity preserved)
    Secondary: Shifted by centroid (brighter presets shift toward white/cyan)
    Tertiary:  Shifted by feliX-Oscar polarity
    """
    r, g, b = engine_accent
    centroid = fingerprint["static"]["centroid_mean"]
    polarity = fingerprint["derived"]["felix_oscar_polarity"]
    warmth = fingerprint["derived"]["warmth"]

    # Primary stays the engine accent
    primary = (r, g, b)

    # Secondary: blend toward white based on centroid (brightness)
    white_blend = centroid * 0.4
    secondary = (
        int(r + (255 - r) * white_blend),
        int(g + (255 - g) * white_blend),
        int(b + (255 - b) * white_blend),
    )

    # Tertiary: feliX-Oscar polarity color
    felix_blue = (0, 166, 214)   # #00A6D6
    oscar_pink = (232, 131, 155) # #E8839B
    t = polarity  # 0 = Oscar, 1 = feliX
    tertiary = (
        int(oscar_pink[0] * (1 - t) + felix_blue[0] * t),
        int(oscar_pink[1] * (1 - t) + felix_blue[1] * t),
        int(oscar_pink[2] * (1 - t) + felix_blue[2] * t),
    )

    return primary, secondary, tertiary
```

### D. Per-Preset Artwork

The MPC expansion browser displays a cover image per program. With fingerprints, each preset can have unique artwork:

```python
def generate_preset_thumbnail(
    engine: str,
    preset_name: str,
    fingerprint: dict,
    output_path: str,
    size: int = 500,
):
    """Generate a unique 500x500 thumbnail for a single preset.

    Smaller than pack cover. No text overlay (MPC shows the name).
    Visual character entirely derived from the fingerprint.
    """
    eng = ENGINE_DEFS.get(engine.upper(), ENGINE_DEFS["DEFAULT"])
    palette = derive_palette_from_fingerprint(eng["accent"], fingerprint)

    # Background: radial gradient using derived palette
    arr = _make_base(size, eng["bg_base"])

    # Central element: the temporal energy curve as a waveform
    energy_curve = fingerprint["temporal"]["energy_curve"]
    draw_energy_waveform(arr, energy_curve, palette[0], size)

    # Scatter density from transient count
    if fingerprint["static"]["transient_density"] > 2.0:
        draw_transient_sparks(arr, fingerprint, palette[1], size)

    # Spectral silhouette (small, centered)
    draw_spectral_silhouette(arr,
        fingerprint["static"]["band_energy_mean"],
        palette[0], size, position="center_bottom")

    # Polarity glow: feliX=blue corona, Oscar=pink corona
    polarity = fingerprint["derived"]["felix_oscar_polarity"]
    draw_polarity_glow(arr, polarity, size)

    img = _arr_to_image(arr)
    img = img.filter(ImageFilter.GaussianBlur(radius=0.5))
    img.save(output_path, "PNG", optimize=True)
```

### E. Collection-Specific Visual Systems

Each planned collection gets a fingerprint-to-visual mapping that reinforces its theme:

**Artwork/Color Collection:**
- XOptic maps the centroid to a position on the visible light spectrum (low centroid = red/infrared, high = violet/ultraviolet)
- The 8-band energies become a prism decomposition: each band maps to a wavelength of light
- Complementary color theory applied: the secondary color is the spectral complement of the centroid color
- Result: presets that sound warm literally look warm (red/orange), presets that sound bright literally look bright (blue/violet)

**Kitchen Essentials Collection:**
- Transient density maps to "texture" -- percussive presets get rough, grainy visual texture (like breadcrumbs or spice grains)
- Energy density maps to "richness" -- dense pads look thick and layered (like a roux)
- Centroid maps to "heat" -- bright presets get warm reds/oranges (spicy), dark presets get cool blues/greens (refreshing)

**Travel/Water Collection:**
- Flux maps to "current speed" -- high flux = rapid visual motion lines (like rapids), low flux = still water
- Band distribution maps to ocean depth: bass-heavy presets get deep ocean colors and pressure distortion, treble-heavy presets get surface shimmer
- The feliX-Oscar polarity maps to water state: feliX = choppy surface (angular cuts), Oscar = calm deep (smooth gradients)


When `fingerprints` is None, behavior is identical to current (static engine styles). When provided, the fingerprint data modulates the style parameters and adds the spectral silhouette element. This means existing callers are unaffected.

### Performance

- Fingerprint analysis of a 4-second stereo WAV at 48kHz = ~384K samples
- Python numpy filter bank: ~200ms per preset on M-series Mac
- For a 150-preset pack: ~30 seconds total fingerprint time
- Per-preset thumbnail generation: ~100ms each (500x500, no text)
- Total additional time for a 150-preset pack: ~45 seconds (fingerprints + thumbnails)

This is negligible compared to the audio render time.
