# XPN Multi-Pack Bundle Strategy — R&D Spec
**Date**: 2026-03-16
**Status**: Draft

---

## Overview

Individual packs are the atomic unit of XO-OX content — one engine, one theme, one sound world. Bundles are the compound unit: a curated grouping of packs that form a complete system. This spec defines bundle types, pricing logic, technical delivery structure, and the first bundle candidates for Q2 2026.

The goal is not to discount — it is to frame. Bundles communicate that these packs belong together and reward customers who commit to the full picture.

---

## 1. Bundle Types

**Engine Bundle** — all packs for a single engine, collected under one roof.
Example: *ONSET Complete* = TIDE TABLES + MACHINE GUN REEF + 2 future ONSET packs. This is the deepest possible investment in one engine's sound world. Best for producers who have adopted an engine as a primary voice.

**Collection Bundle** — all packs spanning one collection arc (Kitchen Essentials, Travel/Water, Artwork).
Example: *Kitchen Essentials Bundle* = all 6 Kitchen collection packs across 6 different engines. This is a cross-engine bundle built around a creative concept rather than a single instrument. Best for producers building a unified production palette.

**Coupling Bundle** — two packs explicitly designed to interact via XOceanus coupling architecture.
Example: *OPAL × OVERDUB* = granular source feeding into tape delay character. Coupling bundles are the most narrative of the three: they exist because the two engines have a documented coupling relationship in XOceanus, and the packs were designed with that signal path in mind.

**Seasonal Bundle** — time-limited cross-collection grab. Black Friday, Pi Day, etc. Deepest discount, shortest window. The contents are curated for the moment (e.g., darkest / most experimental packs), not permanent catalog groupings.

## 2. Bundle Pricing Formula

All prices calculated from sum of individual pack prices (MSRP, not sale price).

| Bundle Type | Multiplier | Discount |
|---|---|---|
| Engine Bundle | × 0.65 | 35% off |
| Collection Bundle | × 0.65 | 35% off |
| Coupling Bundle | × 0.75 | 25% off |
| Seasonal Bundle | × 0.50 | 50% off |

**Floor**: $25 minimum bundle price regardless of formula output. This protects perceived value — a $20 bundle signals a commodity; $25 signals a curated release.

Seasonal bundles apply only during the designated window. Once the window closes, the bundle either disappears from the store or reverts to standard bundle pricing. No permanent "sale" bundles.

## 3. Technical Bundle Structure

Two delivery options were considered:

**Option A**: Single ZIP containing all merged .xpn files flattened together.
Pro: one download, simple. Con: any update requires re-downloading the entire bundle.

**Option B**: Folder ZIP containing separate .xpn files, one per pack, plus a bundle README and `bundle_collection.json`.
Pro: granular updates — only the changed .xpn needs to re-download. Con: customer must load multiple expansions into MPC.

**Recommendation: Option B.**

The MPC workflow already handles multiple expansions natively. Option B aligns with how customers already install packs and allows the content team to update a single pack (bug fix, new preset, remastered samples) without invalidating the entire bundle download. The bundle README bridges the experience by explaining load order, coupling routing suggestions, and which packs interact.

## 4. Bundle Metadata

Each bundle folder contains:

- `bundle_collection.json` at the top level with the following fields:
  ```json
  {
    "name": "ONSET Complete",
    "engines": ["ONSET"],
    "pack_names": ["TIDE TABLES", "MACHINE GUN REEF"],
    "version": "1.0.0",
    "description": "All ONSET packs. Full drum synthesis system."
  }
  ```
- Each individual `.xpn` retains its own `expansion.json` and `bundle_manifest.json` unchanged — packs remain independently installable.
- Bundle cover art: composite grid of individual pack cover arts. 2×2 grid for Coupling and small Engine bundles; 2×3 grid for Collection bundles with 6 packs. Generated by the existing cover art tool in `Tools/`.

Bundle versioning is independent from individual pack versioning. A pack can go from v1.0 to v1.1 without bumping the bundle version, unless the bundle README or `bundle_collection.json` itself changes.

---

## 5. Update Strategy

When a single pack inside a bundle is updated (new presets added, audio bug fixed, sample replaced):

1. The individual `.xpn` version bumps (e.g., `1.0` → `1.1`)
2. The bundle ZIP is rebuilt with the updated `.xpn` swapped in
3. Bundle version bumps minor (e.g., `1.0.0` → `1.0.1`)
4. Customers with the bundle purchase download only the changed `.xpn` file if the storefront supports delta delivery; otherwise the full bundle ZIP re-download is ~same size as before minus the unchanged packs

Customers who purchased the bundle get all pack updates permanently. No upgrade pricing, no re-purchase required. This is the XO-OX social contract: buy it once, it gets better over time.

## 7. Bundle Page Design — XO-OX.org

Bundle pages are not discount pages. The pricing is secondary to the story.

Lead with **why these packs belong together** — the coupling relationship, the collection arc, the creative vision. For ONSET Complete, that story is "one drum synthesis system, four complete sound worlds." For OPAL × OVERDUB, it is "source and character — granular texture meets tape soul."

Show individual pack previews and DNA charts side by side so customers understand what they are getting before they see the price. The visual layout should reinforce the "complete system" frame: these are not random packs on sale, they are a curated toolkit.

The call to action is "complete your system" — not "save 35%."
