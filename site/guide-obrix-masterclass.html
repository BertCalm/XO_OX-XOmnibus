The Brick Philosophy

Every other engine in XOceanus has a fixed identity. ODDFELIX is a snapshot synthesizer. OPAL is granular. ORGANON is variational. You load them and they are what they are: a creature with a fixed body. You can push them hard, layer them with coupling, drive them into strange territory — but the underlying architecture is settled.

OBRIX has no fixed architecture. It is a habitat.

The design philosophy comes from three instruments: the ARP 2600 (semi-modular with normalled routing, 1971), the Serge Modular (“patch-programmable” philosophy, 1974), and the Korg MS-20 (dual sources with dual filters, 1978). All three share the same insight: constraint plus reconfigurability produces more creativity than pure open-endedness. The modular synthesizer gave you infinite routing and produced infinite cognitive load. The semi-modular gave you a sensible default signal path that you could override. That combination — a structure you can break — turns out to be where music happens.

OBRIX inherits this DNA. Its signal path is fixed: two Sources, three Processors, four Modulators, three Effects. What you put in each slot is your choice. Two Saws with two LP filters is a classic subtractive patch. A Noise Source with a Wavefolder and Ring Mod is something else entirely. The constraint is the creativity.

But OBRIX goes beyond the semi-modular ancestors in one crucial way: it has state. The ARP 2600 has no memory of what you played five minutes ago. OBRIX does. Its Stress Memory accumulates velocity over tens of seconds and shifts timbre accordingly. Its Bleach layer remembers sustained high-register play and softens the harmonic ceiling. Its Harmonic Field is a global attractor that progressively tunes every voice toward just-intonation ratios. These are not parameters. They are consequences of how you play.

“The reef doesn’t have a personality. It has a memory. Every coral formation is a record of every current that passed through it.”

OBRIX Engine Commentary, Wave 4

What makes OBRIX different from every other engine is this: you are not programming a sound. You are building an ecosystem and then playing inside it. The distinction sounds abstract until you sit with OBRIX for thirty minutes with Journey Mode on and watch the timbre evolve without any new input from you. That evolution is not random. It is a consequence of the architecture you assembled and the notes you played. The reef is listening.

The Four Brick Sources

OBRIX has two source slots — Source 1 and Source 2 — each choosing from the same menu of nine types. The source types are named in the mythology as Shells: the generators that initiate the signal flow. Before the processors see anything, before the modulators touch anything, the sources define the harmonic raw material.

Off

A source set to Off contributes nothing. Setting both sources to Off produces silence, but the ADSR envelope, modulators, and effects are still running. This matters for coupling: a “silent” OBRIX can still output Brick Complexity data on channel 2 and still receive coupling input into its Reef Residency system. An all-Off OBRIX is not a bypass — it is a dark reef, still biologically active beneath the surface.

Sine, Saw, Square, Triangle — The Classic Four

The four standard waveforms behave exactly as you expect, but OBRIX renders them through PolyBLEP anti-aliasing for the bandlimited types (Saw, Square, Triangle). This means you can run them at very high pitches or through aggressive wavefolder processing without aliasing artifacts folding back into the audible spectrum. The sweet spots:

Sine

Pure Partial

Best as Source 2 tuned to a harmonic interval. Sine has no harmonics to sculpt, so Proc1 LP filter on a Sine is almost pointless. Use Sine for FM modulator material or as a clean fundamental beneath a harmonically complex Source 1.

Saw

Default Starting Point

The OBRIX default. Full harmonic series, all partials present. Excellent for LP filter sculpting. The Wavefolder processor treats Saw input very differently from Square — Saw folds symmetrically, Square folds asymmetrically with more harshness.

Square + Pulse

Only Odd Harmonics

Square contains only odd harmonics, giving a hollow quality that opens dramatically when the Wavefolder is applied. Pulse extends this: obrix_src1PW from 0.05 to 0.95 sweeps from nearly-Sine (very narrow pulse) through Square to an asymmetric buzz. PW modulation via a Modulator slot targeting Pulse Width is the bread-and-butter OBRIX animation technique.

Noise

The Ecology Catalyst

Noise is special in OBRIX because of Wave 4's Brick Ecology. The Symbiosis parameter reads Source 1 amplitude specifically when Source 1 type is Noise (type index 5). With obrix_symbiosisStrength > 0 and Noise in Source 1, the noise amplitude dynamically drives FM depth on Source 2. This is the only source type that participates in cross-source ecology.

Wavetable

Four Banks, Infinite Character

Wavetable mode reads from one of four banks selected by obrix_wtBank: Analog (classic waveform morphs), Vocal (formant-ish transforms), Metallic (spectral spreads and bell partials), Organic (irregular biological waveforms). Wavetable position is modulated by a Modulator slot targeting WT Pos. The Organic bank is the grain-feed material for OPAL coupling.

Driftwood

Irregular Saw Variant

Called “LoFi Saw” internally, Driftwood is an anti-aliased sawtooth with slight phase jitter applied per cycle. The jitter scale is small enough to stay tonal but produces a subtly inconsistent waveform that responds differently to the Harmonic Field — the field “catches” Driftwood with more visible pull because the pitch wanders into the attractor’s range more frequently.

Source FM (obrix_fmDepth, Wave 2) routes Source 1 as a frequency modulator for Source 2. At positive values Source 1 raises Source 2’s frequency; at negative values it lowers it. The range is ±24 semitones of frequency deviation, which at high Source 1 amplitudes becomes dramatic FM territory. The combination of Noise in Source 1 and positive FM depth produces stochastic FM — the Source 2 pitch wanders based on the instantaneous noise value. Combined with the Harmonic Field pulling that wandering pitch toward JI ratios, the result is a probabilistic tuning system that settles into harmony without being programmed to.

Drift Bus Theory

Drift Bus is one of the quieter revolutions in OBRIX. It was added in Wave 3 with two parameters — obrix_driftRate (0.001–0.05 Hz) and obrix_driftDepth (0–1) — and it does something that LFOs fundamentally cannot do: it creates ensemble drift from a single oscillator without periodic repetition becoming audible.

The mechanism: one global ultra-slow LFO runs at a frequency between 0.001 Hz (one full cycle every ~17 minutes) and 0.05 Hz (one full cycle every 20 seconds). Each of OBRIX’s 8 voices reads from this global LFO phase offset by an irrational number multiplied by the voice index. The irrational offsets — based on pi and phi — mean that no two voices share a phase relationship that repeats in any musically relevant timeframe. The voices drift together but never quite identically. The result is the characteristic Berlin School ensemble sound: eight oscillators that breathe as a collective, not as copies.

This is structurally different from detuning (obrix_unisonDetune). Unison detune applies a fixed, static frequency offset to each voice. You can hear the beating between voices because the beating frequency is constant. Drift Bus has no fixed beating relationship between voices. The relative phase is always changing, always irrational, always producing a different interference pattern. Combined with a long Drift Rate (0.002–0.005 Hz), the result sounds organic in a way that traditional LFO detune never achieves.

Drift Parameter Sweet Spots

Very Slow / Deep

Glacial Ensemble

driftRate = 0.003, driftDepth = 0.7. Pitch wandering of up to 35 cents across a five-minute cycle. Barely perceptible at any given moment, but unmistakable over time. Works best with Journey Mode on — the patch breathes like a living system.

Medium / Subtle

Classic Pad Animation

driftRate = 0.01, driftDepth = 0.35. A 100-second cycle with 17-cent pitch wander. This is the sweet spot for pad sounds that feel alive without sounding out of tune. Most OBRIX Atmosphere and Aether presets use this range.

Faster / Moderate

Shimmer and Chorus

driftRate = 0.04, driftDepth = 0.5. A 25-second cycle enters perceptible territory. At this speed, Drift Bus starts to function like a very wide, slow chorus. The phase relationships between voices create a subtle stereo movement even on a mono source if voices are panned.

Combined with Field

Tuned Drift

With obrix_fieldStrength = 0.4 active, the Harmonic Field gently corrects the Drift Bus wander, pulling voices back toward JI ratios as they drift away. The drift creates expressivity; the field prevents it from going out of tune. This combination is unique to OBRIX.

Drift also applies to filter cutoff: at full driftDepth, the global drift LFO modulates cutoff by ±200 Hz per voice with the same irrational phase offsets. This means filter spectral content is slowly shifting independently in each voice, adding to the ensemble texture without any visible LFO motion on a readout. The MOVEMENT macro scales LFO rate across all 4 Modulator slots but does not affect Drift Bus — Drift is intentionally outside the MOVEMENT system, preserving its geological slowness regardless of how much you push the modulation macros.

Journey Macro — The Full Arc

Journey Mode (obrix_journeyMode) is a single toggle that changes OBRIX from a normal synthesizer into something closer to a tape composition tool. When Journey Mode is on, note-off messages are suppressed: the engine ignores key releases. Notes sustain indefinitely, evolving according to the full state machine running underneath.

The arc of a Journey Mode session unfolds in distinct phases, each driven by a different layer of OBRIX’s architecture. Understanding what happens at each stage lets you compose with time rather than just against it.

Phase 0 — Initialization (Note On)

When a note triggers with Journey Mode on, all four Modulator envelopes fire normally, the amplitude envelope attacks through its configured time, and the voice settles into sustain. With eight voices active and sustain levels high, you now have eight independent streams of audio running simultaneously.

Phase 1 — Drift Accumulation (0–30 seconds)

With Drift Bus active, each voice begins its slow departure from unison. The irrational phase offsets ensure they move in different directions at different rates. In the first minute, you hear the ensemble widen from a single pitch center into a lush cloud of closely related pitches. If the Harmonic Field is also active, this widening is shaped — voices drift toward JI ratios rather than wandering arbitrarily.

Phase 2 — Stress Accumulation (30 seconds–2 minutes)

If obrix_stressDecay > 0, the Stress Memory leaky integrator is running. Every note played builds stress. Stress adds up to +900 Hz to the filter cutoff and increases harmonic brightness. In a Journey Mode session where you layer notes over time, stress is the memory of intensity: the reef “tightens” in response to dense playing and “relaxes” during silence. With a 30–60 second time constant, the stress change is perceptible but not instantaneous. You can feel the reef responding to what you played two minutes ago.

Phase 3 — Bleaching (sustained high register)

If obrix_bleachRate > 0, sustained playing above MIDI note 60 (C4) accumulates bleaching. Bleach subtracts up to −700 Hz from filter cutoff and attenuates high harmonics. The bleach recovery time is approximately 5 minutes (the natural recovery constant in the DSP). This mechanic models coral bleaching from thermal stress: sustained bright, high-register play gradually mutes the upper harmonic content. Reef health is a long-arc parameter.

Phase 4 — Ecological Equilibrium

In a long Journey Mode session, stress and bleach approach an equilibrium determined by how you play. Aggressive playing in the high register produces high stress (bright, tight) and accumulated bleach (dark, soft) simultaneously, creating a self-correcting tension. The reef pushes back against extremes. After 5–10 minutes of consistent playing, OBRIX settles into a characteristic timbre that reflects your session’s intensity and register profile. Two different players on the same patch will produce different equilibria. The preset is a starting configuration, not a deterministic output.

Phase 5 — State Reset

The obrix_stateReset trigger (a momentary parameter) clears both stressLevel_ and bleachLevel_ on a rising edge. Triggering this during a Journey Mode session is like opening the windows after a long night — the reef returns to its clean initial state without stopping the notes. This is not an undo; the voices are still playing. It is a systemic refresh, an explicit affordance for managing the reef’s memory as a compositional tool.

Wave 4: The Living Reef

Wave 4 added 14 parameters to OBRIX, transforming it from a sophisticated semi-modular synthesizer into something that had never existed before in the fleet: a synthesis engine with a genuine internal ecology. Four systems interact simultaneously, each contributing a different layer of behavior.

W4 — System 1

Harmonic Field

4 params: fieldStrength, fieldPolarity, fieldRate, fieldPrimeLimit

The Harmonic Field is a per-voice just-intonation attractor/repulsor. Every voice maintains a jifiOffset[2] value in cents for each of its two sources. This offset is updated every sample via a first-order IIR filter converging toward the nearest just-intonation ratio in the selected prime limit.

How it works: Given the current pitch of each voice, the engine computes the nearest ratio in the selected JI table (3-limit Pythagorean, 5-limit extended just, or 7-limit harmonic). The IIR convergence rate is set by obrix_fieldRate (0.001–0.1). At a rate of 0.01, convergence is gentle and takes several seconds to fully settle. At 0.08, voices snap to the JI ratio within a few hundred milliseconds.

fieldPolarity: At 1.0, the field is an attractor — voices are pulled toward JI ratios. At 0.0, it becomes a repulsor — voices are pushed away from JI ratios. Repulsor mode creates controlled microtonality. Combined with Drift Bus, repulsor mode produces pitch material that is perpetually orbiting the harmonic series without ever landing on it.

fieldPrimeLimit choices: 3-limit uses only octaves and fifths (Pythagorean tuning). 5-limit adds major and minor thirds (classical just intonation). 7-limit adds the harmonic seventh and its implications. Each limit produces a different “gravity field” around the note. 7-limit is the most harmonically complex, with the most ratio destinations per octave, and produces the most active voice pull.

W4 — System 2

Environmental Parameters

4 params: envTemp, envPressure, envCurrent, envTurbidity

The four Environmental parameters model the physical medium that sound travels through. These are not metaphors mapped arbitrarily to synthesis parameters — each has a direct, explainable DSP consequence that corresponds to its physical analogue.

Temperature (obrix_envTemp): Scales the Drift Bus depth. Higher temperature means more thermal agitation, more molecular motion, more ensemble drift. At 0.0, temperature has no effect on drift. At 1.0, temperature multiplies the effective drift depth. This makes environmental temperature feel physically accurate: warmer water means more motion.

Pressure (obrix_envPressure): Scales LFO rates across all four Modulator slots via the pressureScale variable. At 0.5 (neutral), pressure has no effect. Above 0.5, LFO rates increase (high pressure creates faster, more agitated modulation). Below 0.5, LFO rates slow down dramatically (deep water, slow timescales). At 0.0, LFO rates are reduced to 30% of their set value.

Current (obrix_envCurrent): A bipolar parameter (−1 to +1) that adds a directional bias to filter cutoff across all voices. At +1, all voices’ cutoffs are pushed up by 2000 Hz (the reef is in a warm current). At −1, all cutoffs are pushed down by 2000 Hz (cold upwelling). Current combined with the Harmonic Field creates a directed drift: warm current simultaneously brightens the filter and biases the JI attractor toward higher-harmonic ratios.

Turbidity (obrix_envTurbidity): Adds per-sample noise to filter cutoff. The noise is computed via a fast linear congruential generator and scaled to ±turbidity×1200 Hz. At low values (0.05–0.1), turbidity adds subtle analog-style filter instability. At high values (0.5+), it becomes a spectral shimmer that can feel like high-frequency noise or alive unpredictability, depending on the source material.

W4 — System 3

Brick Ecology

2 params: competitionStrength, symbiosisStrength

Brick Ecology introduces the first cross-source DSP relationship in OBRIX. Source 1 and Source 2 are no longer independent streams that happen to be mixed — they are now organisms that interact.

Competition (obrix_competitionStrength): Cross-amplitude suppression between Source 1 and Source 2. When Source 1 is loud, Source 2’s amplitude is suppressed toward a 0.1 floor, and vice versa. The suppression amount scales with competitionStrength. At 0.3, the effect is subtle, producing a slight reciprocal duck that gives each source more presence when it dominates. At 1.0, sources fully suppress each other: when Src1 hits its peak, Src2 nearly disappears. Combined with per-source filter processors and different pitch tunings, competition creates a dialogue between two voices rather than a blend.

Symbiosis (obrix_symbiosisStrength): Noise Source 1 amplitude drives FM depth on Source 2. This relationship only activates when Source 1 type is Noise (type index 5). The noise amplitude (a running average of the absolute value of Src1 output) scales the effective FM depth applied to Src2. When noise level is high, Src2 FM index increases, pulling Src2’s pitch wider. When noise settles, FM depth settles. This is a form of amplitude-triggered frequency modulation that has no switch point — it is continuous and amplitude-dependent. The noise feeds the harmonic complexity of its host.

W4 — System 4

Stateful Synthesis

3 params: stressDecay, bleachRate, stateReset

Stateful Synthesis is what makes OBRIX remember. The two accumulators — stressLevel_ and bleachLevel_ — are not per-note parameters. They are engine-global state that persists across notes, across sessions, until a state reset clears them.

Stress Memory (obrix_stressDecay): A leaky integrator with time constant of 30–60 seconds, controlled by stressDecay. The integrator averages velocity across active voices. High velocity notes (hard playing) charge the integrator; silence lets it decay. The current stress level adds up to +900 Hz to filter cutoff and increases the effective harmonic emphasis. stressDecay = 0 turns off the system entirely. At stressDecay = 1.0, the time constant is 60 seconds — a very slow-reacting system that makes the reef remember long patterns of intensity.

Bleaching (obrix_bleachRate): Accumulates from playing above MIDI note 60. Every sample above this register contributes to the bleach level based on how far above C4 the note is (a linear brightness scale). Bleach subtracts up to −700 Hz from filter cutoff and attenuates harmonic content. Recovery is always present at a 5-minute constant regardless of bleachRate; the rate parameter only controls how quickly bleach accumulates. At bleachRate = 0, no bleaching occurs. At 1.0, a sustained high-register chord bleaches the reef noticeably within a few seconds of playing.

State Reset (obrix_stateReset): A momentary trigger that clears both accumulators on the rising edge. Map this to a key, a pad, or a MIDI CC and use it as a compositional device — a reef reset mid-performance. Combined with Journey Mode, state reset becomes a structural tool: you can build stress, listen to the reef change, then wipe it clean and start the accumulation again.

Wave 5: Reef Residency

Wave 5 added just two parameters, but they transform OBRIX’s relationship to the coupling system. Prior to Wave 5, coupling input to OBRIX worked like coupling to any other engine: incoming pitch mod changed pitch, incoming amplitude mod changed filter cutoff. The coupling signal was a control voltage. It was external.

Reef Residency makes coupling ecological. A connected engine doesn’t send a signal to OBRIX. It moves in. OBRIX stores the incoming coupling buffer, measures its amplitude, and treats the connected engine as a third organism in the Brick Ecology system. The four Residency modes define what kind of organism it is.

Off

Default. Coupling input works through the standard pitch mod and filter cutoff routes. No ecological behavior. Backward compatible with all existing presets.

Competitor

The coupling engine’s RMS amplitude suppresses both Source 1 and Source 2 in OBRIX toward a 0.1 floor. When the coupled engine is loud, OBRIX’s own sources back down. When the coupled engine is quiet, OBRIX’s sources recover. At residentStrength = 0.3 (the Guru Bin default sweet spot), this creates a gentle reciprocal relationship: OBRIX defers to its noisy neighbor.

Competitor mode is the right choice when OBRIX is a background texture and the coupled engine is a melodic foreground. The coupling not only sends audio downstream — the coupled engine’s presence automatically ducks OBRIX without any sidechaining setup.

Symbiote

The coupling engine’s amplitude drives two things simultaneously: it increases effective FM depth on OBRIX’s sources, and it boosts the Harmonic Field strength. The louder the symbiote, the more harmonically complex OBRIX becomes, and the more aggressively the JI attractor pulls its voices. There is a mutual benefit: OBRIX responds to the symbiote’s presence by becoming more harmonically sophisticated.

Symbiote mode is the right choice for an OBRIX + OPAL pairing with the Reef Memory configuration. OPAL’s granular output returns into OBRIX as a symbiote, boosting the Harmonic Field and making OBRIX’s voices tune more aggressively to JI ratios. The relationship becomes self-reinforcing: cleaner OBRIX output makes better grain material for OPAL, which returns a stronger signal, which makes OBRIX more harmonically coherent.

Parasite

The coupling engine’s energy feeds OBRIX’s stress and bleach accumulators. The full-band RMS drives stress level; the high-frequency content above 2kHz drives bleach level. A coupled engine playing loud, bright material will steadily tighten OBRIX’s filter and eventually soften its harmonic ceiling — even if OBRIX’s own note velocity is low.

Parasite mode is the right choice when you want the coupled engine to literally change OBRIX’s character over time. Route a drum machine into OBRIX as a Parasite: snare transients are high-frequency, they accumulate bleach rapidly. Kick drums are low-frequency, they accumulate stress. After 30 seconds, OBRIX’s timbre is a portrait of the drum pattern that fed it.

Three Tutorial Patches

Each tutorial patch corresponds to a .xometa preset file in the XOceanus preset library. The parameter values below are the exact values stored in those files. Build them manually to understand the design decisions, or load the presets and use this text as an explanation of what you’re hearing.

Tutorial Patch 01

Single Brick Foundation

TUTORIAL_OBRIX_01_Single_Brick_Foundation.xometa — Foundation mood

The most essential OBRIX configuration: one source, one processor, one modulator. Everything else is default or off. The goal is to demonstrate that OBRIX with minimal bricks is a complete, musical instrument — and to establish a reference point for understanding what each subsequent layer adds.

// Source 1 — the only generator obrix_src1Type = Saw // classic sawtooth: full harmonic series obrix_src2Type = Off // no second source obrix_srcMix = 1.0 // Src1 fully dominant // Processor 1 — LP filter on Src1 obrix_proc1Type = LP Filter obrix_proc1Cutoff = 2200 Hz // warm, focused midrange obrix_proc1Reso = 0.22 // slight resonance at the cutoff obrix_proc2Type = Off obrix_proc3Type = Off // Amp envelope obrix_ampAttack = 0.01s obrix_ampDecay = 0.4s obrix_ampSustain = 0.75 obrix_ampRelease = 0.55s // Modulator 1 — filter envelope obrix_mod1Type = Envelope obrix_mod1Target = Filter Cutoff obrix_mod1Depth = 0.55 // envelope sweeps filter up on attack obrix_mod2Type = Velocity obrix_mod2Target = Volume obrix_mod2Depth = 0.65 // velocity shapes loudness (D001) // No biophonic systems, no drift, no Wave 4 obrix_driftDepth = 0.0 obrix_fieldStrength = 0.0 obrix_polyphony = Poly8 obrix_level = 0.8
What to Listen For

Play a sustained chord — a simple triad works. Notice the filter envelope opening on each attack: the bright transient followed by the settled timbre. Now push the CHARACTER macro up to 0.5 while holding notes and hear the wavefolder engage on the filter output, adding harmonics. This is the single-brick foundation: a complete instrument from five parameters, with the macro system ready to take it further.

Tutorial Patch 02

Drift Bus Demo

TUTORIAL_OBRIX_02_Drift_Bus_Demo.xometa — Atmosphere mood

This patch is specifically designed to make the Drift Bus audible. Eight voices are stacked at the same pitch with Journey Mode on and a very slow drift rate. Hold a single note and wait. The ensemble effect emerges gradually over 20–40 seconds as the irrational phase offsets between voices develop into audible differences. At 90 seconds, the patch sounds like eight different oscillators even though you played one note.

// Sources — dual Saw for rich ensemble material obrix_src1Type = Saw obrix_src2Type = Saw obrix_src2Tune = +0.07st // minimal static detune to seed the drift obrix_srcMix = 0.5 obrix_fmDepth = 0.0 // no FM — keep sources independent // One LP filter on the mixed output (Proc3) obrix_proc1Type = Off obrix_proc3Type = LP Filter obrix_proc3Cutoff = 5800 Hz obrix_proc3Reso = 0.15 // Drift Bus — the star of the show obrix_driftRate = 0.008 // 125-second cycle obrix_driftDepth = 0.65 // substantial drift range obrix_journeyMode = 1 // CRITICAL: suppress note-off // Sustained envelope — infinite sustain obrix_ampAttack = 0.8s // slow attack builds the cloud gradually obrix_ampDecay = 0.2s obrix_ampSustain = 1.0 // full sustain (Journey keeps it running) obrix_ampRelease = 8.0s // Spatial obrix_distance = 0.35 // distant placement, HF rolloff obrix_air = 0.35 // slightly warm air tilt obrix_unisonDetune = 3 ct // very subtle static detune beneath the drift obrix_polyphony = Poly8
What to Listen For

Hold one note and do not touch any other control. At 10 seconds, the attack has finished and you hear a slightly wide single pitch. At 40 seconds, the eight voices have drifted apart enough to produce audible beating and chorusing. At 90 seconds, the drift has created an interval structure between voices that was not programmed — it emerged from irrational phase math. The sound has become polyphonic from a monophonic input. This is what Berlin School composers discovered with their tape-delay ensemble systems. OBRIX produces it from one key press.

Tutorial Patch 03

Journey Arc

TUTORIAL_OBRIX_03_Journey_Arc.xometa — Flux mood

This patch has all Wave 4 stateful systems active. The goal is to demonstrate the full Journey arc: play aggressively in the high register for 2 minutes, then shift to quiet playing in the low register. The reef will visibly respond. Monitor the filter character while you play — watch it tighten, brighten, then slowly soften as the ecosystem remembers your history.

// Sources — Noise + Saw for ecology demonstration obrix_src1Type = Noise // Noise enables Symbiosis ecology obrix_src2Type = Saw obrix_srcMix = 0.28 // Noise quieter, Saw dominant // Processors obrix_proc2Type = BP Filter // bandpass on noise: resonant noise band obrix_proc2Cutoff = 3400 Hz obrix_proc2Reso = 0.55 obrix_proc3Type = LP Filter // post-mix low-pass obrix_proc3Cutoff = 6000 Hz obrix_proc3Reso = 0.1 // Journey Mode + Drift obrix_journeyMode = 1 obrix_driftRate = 0.012 obrix_driftDepth = 0.4 // Harmonic Field — 7-limit attractor obrix_fieldStrength = 0.5 obrix_fieldPolarity = 1.0 // attractor mode obrix_fieldRate = 0.02 obrix_fieldPrimeLimit = 7-Limit // richest JI gravity // Environmental obrix_envTemp = 0.45 // moderate temperature: some thermal drift obrix_envPressure = 0.65 // slightly elevated: LFOs run fast obrix_envTurbidity = 0.15 // subtle spectral shimmer // Brick Ecology obrix_symbiosisStrength = 0.5 // noise drives FM on Saw // Stateful Synthesis — the memory systems obrix_stressDecay = 0.55 // 46-second stress time constant obrix_bleachRate = 0.4 // moderate bleach accumulation obrix_polyphony = Poly8
What to Listen For

Play hard in the high register for 90 seconds — repeated chords above C4, hard velocity. Notice the filter brightening and tightening as stress accumulates. Then switch to quiet, low-register long notes. Two things happen over the next minute: the stress level decays, softening the brightness; and the bleach you accumulated from the high-register playing begins to show (though bleach recovery takes 5 minutes). The patch is a geological portrait of how you played it. Finally, trigger State Reset (map to a pad or key) and hear the reef clear itself while the notes continue to sustain.

The OBRIX Coupling Cookbook

These three pairings are the starting points documented in the Coupling Cookbook: OBRIX Pairs post. Each is summarized here as a quick reference. The full post includes complete parameter listings and step-by-step setup instructions.

OBRIX + OPAL AudioToBuffer

Reef Memory

OBRIX’s biophonically shaped output streams into OPAL’s granular buffer as the exclusive grain source. OPAL grains crystallize instants of OBRIX’s harmonic field and scatter them across time. Set OBRIX Reef Residency to Symbiote — OPAL’s returning audio boosts OBRIX’s Harmonic Field, creating a self-reinforcing loop: cleaner OBRIX output becomes better grain material. Best mood: Entangled.

OBRIX + ONSET AmpToChoke

Percussive Ecology

OBRIX’s amplitude chokes ONSET’s voices: when OBRIX is loud, ONSET drums are silenced. When OBRIX decays, ONSET breathes. The result is a sidechain relationship without a compressor — purely ecological. Set ONSET as OBRIX Reef Residency Competitor: ONSET’s hits suppress OBRIX’s sources during transients, preventing mud from both sides simultaneously. Best mood: Entangled / Submerged.

OBRIX + OVERDUB AmpToFilter

Reef Echo

OBRIX’s environmental state propagates into OVERDUB’s filter character. Specifically, OBRIX’s amplitude envelope opens and closes OVERDUB’s filter cutoff — OVERDUB’s echo tail blooms or tightens in sympathy with the reef. Set OBRIX Reef Residency off for this pairing; OVERDUB’s delay returns are deliberately kept outside the ecology. The coupling is one-directional: the reef shapes the echo, the echo does not shape the reef. Best mood: Atmosphere / Aether.

The coupling system documentation at The Space Between the Engines covers all 15 coupling types. OBRIX is compatible with AudioToFM, AmpToFilter, LFOToPitch, AmpToPitch, and AmpToChoke on the input side. On the output side, it exports audio on channels 0 and 1, and Brick Complexity (a 0–1 signal encoding how many bricks are currently active) on channel 2. That complexity signal is unique to OBRIX: no other engine exports architectural metadata as a coupling signal. A downstream engine can modulate its own behavior based on whether OBRIX is configured with one brick or eight.
