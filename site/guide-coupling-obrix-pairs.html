Coupling Cookbook: OBRIX Pairs

Three recipes showing OBRIX coupled with OPAL, ONSET, and OVERDUB. When you route a signal to or from a biophonic engine — one that tracks state, accumulates stress, responds to competition — you are not coupling to a static oscillator. You are coupling to a living system.

CategoryArchitecture — Coupling Recipes
Read Time18 min
EnginesOBRIX, OPAL, ONSET, OVERDUB
AccentReef Jade #1E8B7E

Every coupling guide in XOlokun eventually runs into the same question: which engine should go in which slot, and why does the choice matter beyond signal routing? With most pairings the answer is practical — this engine has a good filter cutoff range to be modulated, that engine has a stable LFO worth stealing. The coupling is a technical connection between two independent systems.

OBRIX is different. It is not an instrument that produces sound from a fixed architecture. It is a habitat. The reef. Underneath the brick-selection UI and the modulator slots is a living system with four distinct layers of internal state: a Harmonic Field that exerts magnetic pressure on its own voices' tuning, an Environmental layer that models temperature and pressure and current and turbidity as synthesis parameters, a Brick Ecology where Source 1 and Source 2 compete or cooperate for amplitude, and a Stateful Synthesis layer that accumulates and releases stress over tens of seconds. On top of all of this, Wave 5's Reef Residency mode makes coupling itself ecological: a connected engine doesn't receive a signal from OBRIX — it moves into OBRIX.

This means that coupling to OBRIX has consequences that don't exist with other pairings. When OPAL streams audio into OBRIX's buffer, OBRIX's Reef Residency system receives OPAL as a new organism in the Brick Ecology: a Symbiote whose amplitude boosts the Harmonic Field and drives extra FM depth. When ONSET is downstream of OBRIX, OBRIX's amplitude doesn't just modulate a parameter — it chokes or releases ONSET's voices the way a dominant species controls when its neighbors breed. When OVERDUB is linked by filter, OBRIX's environmental state — its temperature, its pressure, its current direction — propagates directly into OVERDUB's decay character.

None of this is metaphor. It is exactly what the DSP does.

The Three Recipes

These recipes correspond to the three Entangled presets in Presets/XOlokun/Entangled/: Reef Memory, Percussive Ecology, and Reef Echo. Each can be loaded directly, but the setup instructions below let you build them from scratch and understand why each parameter choice was made.

OBRIX + OPAL

Recipe 1 — Reef Memory

Coupling type: AudioToBuffer

OBRIX's brick-constructed waveshapes stream directly into OPAL's granular buffer. The signal OPAL grains read from is not a static sample or an internal oscillator — it is the live output of a biophonic reef engine that is evolving in real time. OPAL reads back from a system that has already been shaped by Harmonic Field correction, competition between sources, temperature-scaled drift, and the Organic wavetable bank. Every grain you hear is a fragment of that architecture, scattered and re-scattered through OPAL's position and pitch scatter parameters.

The key insight is opal_externalMix = 1.0: OPAL is set to use only the incoming AudioToBuffer signal as its grain source, not its own oscillators. This is the Reef Memory configuration — OPAL as pure reader of OBRIX's accumulated state. The granular engine becomes the reef's memory, crystallizing instants of the harmonic field and playing them back at a density and position of your choice.

OBRIX Setup (Slot 0)
// Sources obrix_src1Type = Saw // harmonic-rich source for grain material obrix_src2Type = Sine // upper harmonic partial, tuned +7st obrix_src2Tune = +7.0 // perfect fifth above Src1 obrix_srcMix = 0.62 // Src1 dominant // Processors obrix_proc1Type = LP Filter obrix_proc1Cutoff = 3200 Hz // midrange warmth before granulation obrix_proc1Reso = 0.28 obrix_proc2Type = Wavefolder // harmonic fold adds grain texture // Biophonic obrix_wtBank = Organic // wavetable bank 3 — best grain material obrix_fieldStrength = 0.45 // JI attractor active — tunes itself obrix_driftDepth = 0.35 // Berlin School per-voice drift obrix_stressDecay = 0.25 // stress memory shapes timbre over time obrix_unisonDetune = 8 ct // subtle ensemble spread // Reef Residency obrix_reefResident = Symbiote // OPAL's audio boosts OBRIX field strength obrix_residentStrength = 0.3
OPAL Setup (Slot 1)
opal_externalMix = 1.0 // CRITICAL: use AudioToBuffer input exclusively opal_couplingLevel = 0.82 // how much of OBRIX's audio enters the buffer opal_grainSize = 0.38 // medium grains — long enough to hear harmonic content opal_density = 0.55 // moderate cloud density opal_posScatter = 0.42 // moderate position randomization opal_pitchScatter = 0.08 // slight pitch scatter — stays tonal opal_panScatter = 0.35 // wide stereo grain field opal_shimmer = 0.28 // shimmer tail extends the crystallization opal_ampAttack = 0.22s // slow grain cloud attack opal_ampRelease = 2.0s // long tail
Coupling Route
Source: Obrix → Target: Opal Type: AudioToBuffer Amount: 0.15 // Guru Bin sweet spot: audible but not overwhelming
Step by Step
  1. 1Load OBRIX in slot 0. Set Source 1 to Saw, Source 2 to Sine, tune Source 2 up +7 semitones. Set Wavetable Bank to Organic.
  2. 2Load OPAL in slot 1. Set opal_externalMix to 1.0 — this is the critical step. OPAL will now read its grain buffer from the AudioToBuffer coupling input instead of its own oscillators.
  3. 3Set the coupling route: OBRIX → OPAL, type AudioToBuffer, amount 0.15.
  4. 4On OBRIX, enable Reef Residency: Symbiote at strength 0.3. Now OPAL's audio feeding back into OBRIX boosts its Harmonic Field — the two engines begin to tune toward each other.
  5. 5Play a sustained chord on OBRIX. Give it five seconds. The Harmonic Field attractor is pulling voices toward just-intonation ratios. The grains OPAL reads are already tuned — the crystallization is musical, not random.
What to listen for

The moment the Harmonic Field has settled, the grain cloud from OPAL will have a distinct tonal character — warmer and more coherent than granulation of a static source would sound. Play new notes. Notice that the grain cloud lags the notes by the granular attack time, creating an automatic call-and-response where OBRIX's current state and OPAL's memory of OBRIX's past state overlap. With opal_posScatter = 0.42, grains are reading from slightly different positions in the buffer, smearing the architectural detail into something crystalline.

Go further

Try enabling OBRIX's Journey Mode (obrix_journeyMode = 1). Notes no longer release — OBRIX sustains and evolves indefinitely. The buffer OPAL reads from becomes a continuously updating archive of the reef's biophonic state. Enable OPAL's Freeze at a resonant moment: the granulator locks into a single OBRIX state, crystallized permanently.

OBRIX + ONSET

Recipe 2 — Percussive Ecology

Coupling type: AmpToChoke

When OBRIX's amplitude is high — when a chord is being held, when the biophonic harmonic field is active — ONSET drum hits are choked. When OBRIX goes quiet, the percussion breathes. This is an ecological relationship: the dominant harmonic organism suppresses the percussive surface species. In a natural coral reef, large organisms create acoustic shadows that determine when smaller organisms can vocalize.

The crucial OBRIX configuration for this recipe is Brick Ecology active: Competition Strength at 0.42, Symbiosis at 0.28. With competition enabled, OBRIX's two sources — Saw and Noise — are suppressing each other's amplitude at a low level. This creates an internal pulsing, an ecological heartbeat in the Brick layer, whose RMS envelope drives the AmpToChoke routing to ONSET. OBRIX doesn't just send a steady amplitude signal — it sends a signal that has been shaped by internal competition, which creates rhythmically irregular choking behavior rather than a simple gate.

OBRIX Setup (Slot 0)
// Sources — Saw + Noise creates ecological tension for Brick Ecology obrix_src1Type = Saw obrix_src2Type = Noise obrix_srcMix = 0.7 // Saw dominant // Brick Ecology — this is the ecological engine of the choke behavior obrix_competitionStrength = 0.42 // Saw vs Noise amplitude suppression obrix_symbiosisStrength = 0.28 // Noise → FM on Saw (noise nurtures tone) // Environmental — turbidity adds noise that shapes envelope character obrix_envTemp = 0.35 // thermal drift amplifies ecological variation obrix_envPressure = 0.65 // LFO rate scaling > neutral obrix_envTurbidity = 0.12 // spectral noise makes envelope irregular // Reef Residency — ONSET becomes a Competitor organism obrix_reefResident = Competitor obrix_residentStrength = 0.35
ONSET Setup (Slot 1)
// ONSET's kit runs normally; the choke is applied by coupling perc_macro_punch = 0.65 // punchy kit — contrast makes choking visible perc_xvc_hat_choke = 0.55 // ONSET's internal hi-hat choke also active perc_xvc_global_amount = 0.4 // XVC active — kit already has internal ecology perc_macro_space = 0.38 // relatively dry — contrast with OBRIX's reverb
Coupling Route
Source: Obrix → Target: Onset Type: AmpToChoke Amount: 0.14 // enough to choke clearly; not so much that drums disappear entirely
Step by Step
  1. 1Load OBRIX in slot 0 and ONSET in slot 1.
  2. 2On OBRIX, set Source 2 to Noise. Enable Competition at 0.42 and Symbiosis at 0.28. This is the critical step: without Brick Ecology active, the AmpToChoke signal will be smooth and regular. With ecology, it pulses with the internal organism competition.
  3. 3Set Temperature to 0.35 and Turbidity to 0.12. This adds thermal noise to the internal state, making the choke signal organic rather than mechanical.
  4. 4Set Reef Residency to Competitor, strength 0.35. ONSET's audio arriving back through the coupling bus now cross-suppresses both of OBRIX's sources, creating a bidirectional ecological relationship.
  5. 5Set the coupling route to AmpToChoke, amount 0.14. Play a sustained chord in OBRIX while triggering the ONSET kit. The drum hits will be strangled by the chord's presence.
  6. 6Release the OBRIX chord. The percussion opens up immediately. Now stab a short OBRIX note while keeping a rhythmic kick pattern going — the stab will briefly silence the kit, then the kick re-enters as OBRIX releases. This is the call-and-response ecology.
What to listen for

Because OBRIX's Competition and Turbidity parameters are creating an uneven amplitude envelope, the choking behavior is not a clean gate — it has an organic, slightly ragged edge. Drums don't disappear cleanly; they are suppressed, breathe briefly, get suppressed again. This matches the behavior of organisms in an ecosystem: the dominant species doesn't eliminate the smaller ones, it just makes their survival difficult when it is actively present.

Go further

Map OBRIX's COUPLING macro to a fader or expression pedal. At macro 0, the coupling route is at its minimum effect and the drums play freely. As you push the macro, the choke tightens. At macro 1.0 with high OBRIX amp, the drums are nearly completely suppressed. This gives you live ecological control over the relative weight of harmonic and percussive material — a technique no mixer channel can provide because the coupling is not a volume relationship; it is an amplitude-dependency relationship.

OBRIX + OVERDUB

Recipe 3 — Reef Echo

Coupling type: FilterToFilter

OBRIX's filter output — shaped by its Environmental layer's current, pressure, and temperature — feeds directly into OVERDUB's filter input. When OBRIX's reef is warm (high obrix_envTemp), its filter cutoff rises, and that rise propagates into OVERDUB's filter as well: OVERDUB brightens in sympathy. When OBRIX drifts toward its negative current bias (obrix_envCurrent = -0.12), both engines darken together. The two engines share an environmental condition.

OVERDUB's tape delay and spring reverb are particularly interesting here because their saturation character depends on the signal level feeding them — which is now controlled by the coupled filter. When OBRIX's filter sweeps open wide, OVERDUB's delay input gets a brighter signal, which sits differently in the feedback bandpass than a darker input would. The tape is not just copying OVERDUB's notes; it is responding to OBRIX's ecology.

OBRIX Setup (Slot 0)
// Source — Organic wavetable + Sine sub for a rich filter source obrix_src1Type = Wavetable obrix_wtBank = Organic obrix_src2Type = Sine obrix_src2Tune = -12.0 // sub octave Sine for bass body obrix_unisonDetune = 12 ct // wide unison for spatial breadth // Environmental — this is what propagates to OVERDUB obrix_envTemp = 0.28 // warm environment — increases drift amplitude obrix_envPressure = 0.45 // slightly below neutral — LFO rate damped obrix_envCurrent = -0.12 // directional current: -200 Hz cutoff bias obrix_envTurbidity = 0.08 // subtle spectral noise // Stateful Synthesis — stress and bleach create long-arc filter movement obrix_stressDecay = 0.35 // velocity stress → +900 Hz cutoff over minutes obrix_bleachRate = 0.15 // high-register play → −700 Hz (darkens over time) // Reef Residency — OVERDUB as Parasite obrix_reefResident = Parasite // OVERDUB energy feeds OBRIX's stress accumulator obrix_residentStrength = 0.28
OVERDUB Setup (Slot 1)
// OVERDUB as lo-fi texture complement to OBRIX's wavetable character dub_oscWave = Saw dub_subLevel = 0.35 // sub presence for warmth dub_drift = 0.25 // analog drift — both engines drift; OBRIX's is biophonic dub_filterCutoff = 3200 Hz // starting point before coupling modulation dub_filterReso = 0.22 // slight resonance makes sweeps audible // Send/Return — tape and spring are the texture dub_sendLevel = 0.55 // meaningful send into tape delay dub_returnLevel = 0.65 // return mix brings echo forward dub_driveAmount = 2.2 // mild tape saturation dub_lfoRate = 0.15 Hz // very slow LFO — breathing-rate filter movement dub_lfoDepth = 0.18 dub_lfoDest = Filter // LFO on filter; coupling is also on filter — both active
Coupling Route
Source: Obrix → Target: Overdub Type: FilterToFilter Amount: 0.16 // enough for environmental changes to be audible in OVERDUB
Step by Step
  1. 1Load OBRIX in slot 0 with the Organic wavetable bank. Set Environmental parameters: Temperature 0.28, Current -0.12. These are the weather conditions both engines will share.
  2. 2Enable Stress Decay at 0.35 and Bleach Rate at 0.15. The filter cutoff will shift by up to ±900 Hz over the course of your session depending on how aggressively you play and how many high notes you hit. This long-arc movement propagates into OVERDUB via the filter coupling.
  3. 3Load OVERDUB in slot 1. Set its LFO destination to Filter. Both OVERDUB's own slow LFO and the incoming OBRIX filter coupling will modulate the filter, creating interference between a musical slow LFO and an environmental slow modulation. These do not always align, which is the point.
  4. 4Set Reef Residency to Parasite. OVERDUB's high-frequency energy feeding into OBRIX's coupling bus will accumulate in OBRIX's stress level, which over time darkens OBRIX's filter via the bleach accumulator. The two engines are aging each other.
  5. 5Play the same chord repeatedly for two minutes while listening to OVERDUB's echo character. It will not sound the same at minute two as it did at minute zero — stress and bleach have accumulated and the coupled filter is darker.
What to listen for

In the first 30 seconds, OVERDUB's tape echo has a clean, relatively bright character because OBRIX's stress level is near zero. Play hard and fast for a minute — lots of high notes, high velocity. By the two-minute mark, OBRIX has accumulated stress and bleach, its filter is darker, and that darkness has propagated to OVERDUB. The tape echo is now murkier. The same playing technique produces different sonic results depending on how long the session has been running. This is not automation. This is stateful synthesis coupling.

Go further

After the session has aged OBRIX's state, trigger the State Reset parameter (obrix_stateReset) with a momentary press. Stress and bleach levels clear to zero instantly. OVERDUB's filter brightens back to its starting character. You now have a session "refresh" gesture — the ability to reset the ecological age of the pair. Assign this to a MIDI note, a macro, or a hardware button for live use.

Why OBRIX Pairs So Well

Most coupling guides in XOlokun describe what happens when two signals interact. OBRIX requires a different framing: what happens when two systems interact. The difference is not poetic. It is architectural.

Reef Residency

The Third Organism

Wave 5's Reef Residency mode adds the coupled engine as a third ecological organism in OBRIX's internal Brick Ecology. As a Competitor, the coupled engine's amplitude suppresses both of OBRIX's sources. As a Symbiote, its amplitude boosts the Harmonic Field and drives FM depth. As a Parasite, its high-frequency energy feeds the stress and bleach accumulators over time. No other engine in the fleet has this. The coupling is not a one-way parameter modulation — it is a mutual ecological enrollment.

Biophonic State

The Narrative Signal

When you couple from OBRIX to another engine, the signal the target receives has already been shaped by OBRIX's internal state — by how long the session has been running, by how aggressively it has been played, by the current values of temperature and pressure and current in the Environmental layer. The target is not receiving a static oscillator output. It is receiving a narrative: the acoustic record of how OBRIX has been living. AudioToBuffer coupling to OPAL makes this literal: OPAL reads fragments of that narrative as grain material.

Harmonic Field

Tuning by Proximity

OBRIX's Harmonic Field exerts a just-intonation attractor or repulsor force on its own voices. When another engine is coupled in as a Symbiote, the Harmonic Field is boosted by the coupling amplitude — which means the coupled engine's activity actually tightens OBRIX's own intonation. The downstream engine is making OBRIX more itself. This feedback loop — coupling input improving the source engine's core behavior — is only possible with an engine that has internal dynamic attractor states.

5 Sources × 3 Processors

Timbral Survival

OBRIX has 9 source types across 2 source slots, 6 processor types across 3 processor slots, 4 modulator slots, and 3 FX slots with variable routing. This is the richest timbral palette in the fleet. When coupling is applied to or from OBRIX — especially at higher amounts — the engine has enough internal parameter space to absorb the modulation without losing its character. Its identity survives heavy coupling because it has so many dimensions of variation that some always remain untouched by the incoming signal.

Wave Architecture

Built in Layers

OBRIX was designed in five waves, each adding a layer of ecological complexity. The result is an engine where the biophonic layer (Wave 4) and the coupling layer (Wave 5) were designed in sequence, with Wave 5 explicitly aware of Wave 4's systems. Reef Residency was built to interact with Brick Ecology, Harmonic Field, and Stateful Synthesis simultaneously. This layered design means coupling behaviors are not emergent accidents — they are architecturally planned interactions between named, documented systems.

The Blessing

B016 Amended

OBRIX's seance resolved a foundational debate: can synthesis-layer interdependence between voices (shared JI attractor, cross-voice amplitude ecology, environmental globals) coexist with MIDI-layer voice independence? B016 Amended said yes — provided no synthesis-layer coupling propagates back to affect MIDI routing or voice stealing. This permission is what makes OBRIX's internal ecology architecturally coherent. The reef breathes as one organism without compromising the independence of the notes that play through it.

General Principles

A few rules that apply to any OBRIX pairing, not just these three:

  • Set Reef Residency before finalizing coupling amounts. The mode — Competitor, Symbiote, Parasite — changes the semantic meaning of the coupling amount. At Competitor mode, a higher coupling amount suppresses OBRIX's sources more. At Symbiote mode, it boosts the Harmonic Field more. Dialing in the amount without knowing the mode first will give you inconsistent results across sessions.
  • Activate at least one biophonic layer. Coupling to OBRIX with all biophonic parameters at zero is like coupling to a conventional subtractive synth — technically functional but missing the point of the engine. At minimum, set obrix_fieldStrength to 0.2–0.5 and obrix_stressDecay to 0.1–0.3. These are the two parameters most audible in coupling contexts because they directly shape the amplitude envelope character over time.
  • Expect coupling behavior to change over the session. Stress accumulates. Bleach accumulates. The Harmonic Field converges and diverges. An OBRIX pairing at minute one will sound different at minute ten, and different again at minute twenty. This is not drift or instability — it is designed ecological aging. Plan for it, or use the State Reset as a session boundary marker.
  • The Guru Bin sweet spot for coupling amount is 0.12–0.18. Below 0.12, biophonic coupling routes are inaudible in most contexts. Above 0.20, the ecological relationships become dominant enough to destabilize the secondary engine's character. The range 0.13–0.17 gives you clearly audible coupling that still lets the secondary engine retain its identity.
